Authentication

  • Login options

    At first, I was unsure about the login options I wanted in the app, then decided that an Anonymous login wasn't the way to go about this app, since many features here rely on working with an authenticated user one way or another. Hence finally after long hours of brainstorming, I decided on three options Google, Phone number, and Email. These options will be used to authenticate a user.
    Old New

  • OTP based Login

    The OTP based login is based on Google's api for SMS but the limitation is that the length of OTP is fixed to six. So therefore I have wrapped the otp boxes in a 'Wrap' widget.
    Old New

  • Email based Login

    EmailJs api is used to send email to send OTP, in this case I have also fixed the length of the OTP to six for consistency for the SMS based login.
    Old New

  • Google based Login

    The google based login was easy to implement with firebase on android and such, but for web i had to make some changes in the manifest file to properly implement the google login.

    The home page is just a dummy page with an expansion tile and hard-coded questions. So there is not much to look at here. It is just so that one can explore the search engine. So it is just for show for now.
